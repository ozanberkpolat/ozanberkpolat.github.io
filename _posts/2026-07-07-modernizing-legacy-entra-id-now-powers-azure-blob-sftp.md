**Stop managing local SFTP users like it's 1999.** For years, the trade-off for using the ubiquitous SFTP protocol was the administrative headache of managing local credentials and SSH keys outside of your primary identity provider. In a Zero Trust world, these "identity silos" represent a significant security risk and a massive drain on operational efficiency.

The game has officially changed. Microsoft has announced the **General Availability of Entra ID-based access for Azure Blob Storage SFTP**. This update allows you to ditch local passwords in favor of centralized, enterprise-grade security. By leveraging Entra ID identities—including guest users via Entra External Identities—you can now secure your file transfers with the same robust policies that protect the rest of your Azure environment. It is the perfect fusion of legacy protocol support and modern identity management, available now across all regions.

Start securing your data transfers today.
