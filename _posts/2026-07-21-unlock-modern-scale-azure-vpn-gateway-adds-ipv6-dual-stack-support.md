The transition to **IPv6** is no longer a distant roadmap item—it is a critical necessity for modern, scalable cloud architectures. As IPv4 addresses become increasingly scarce and expensive, Azure is empowering enterprises to modernize their hybrid connectivity. The latest update introduces official support for **IPv6 inner traffic** within Azure VPN Gateway, enabling a seamless dual-stack configuration.

This high-impact update is available across all production SKUs utilizing standard public IPs. Whether you are managing complex **Site-to-Site (S2S)** tunnels for global branch offices or secure **Point-to-Site (P2S)** connections for a distributed workforce, you can now tunnel IPv6 traffic alongside traditional IPv4. This flexibility allows for a phased migration to next-generation networking without disrupting your existing infrastructure or security posture.
